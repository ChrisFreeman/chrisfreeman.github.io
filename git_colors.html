Git Colors

Git colors can be customized through the command-line interface (CLI) or through editing the git config files.

This information was gleaned from git-scm.com

Git Config files

Git config files affect repositories at three levels. Local levels have precedent over more global levels

/etc/gitconfig "--system" wide configuration
~/.gitconfig "--global" configuration across all user repositories
.git/config "--local" configuration of single repository (default)

Git Config CLI

Git config files can be updated on the command line using git config and specifying the git config file-option.

git config [<file-option>] name value

Enabling Colors

color.ui: false (or never) | always | true (or auto)

Set it to true or auto (this is the default since Git 1.8.4) to use color when written to the terminal. For example,

$ git config --global color.ui true

Specifying Colors

Git colors are specified as <foreground> <background> <attributes>

Colors: normal | black | red | green | yellow | blue | magenta | cyan | white | [0-255] | #[000000-ffffff]

ANSI 256 color and 24-bit RGB color as hex are terminal dependent.

Attributes on: bold | dim | ul | blink | reverse
Attributes off: nobold | nodim | noul | noblink | noreverse.

git-branch

git-branch enable/disable color

color.branch:  always | false (or never) | auto (or true)

Auto (or true): colors are used only when the output is to a terminal. Defaults to false.

git-branch color customize <slot>

color.branch.<slot> where <slot> is
    current (the current branch) |
    local (a local branch) |
    remote (a remote-tracking branch in refs/remotes/) |
    upstream (upstream tracking branch) |
    plain (other refs)

git-diff, git-log, git-show

git-diff enable/disable color to patches

color.diff:  always | false (or never) | auto (or true)

Always, git-diff, git-log, and git-show will use color for all patches. If true or auto, those commands will only use color when output is to the terminal. Defaults to false.

git-diff color customize <slot>

color.diff.<slot> where <slot> is
    plain (context text) |
    meta (metainformation) |
    frag (hunk header) |
    func (function in hunk header) |
    old (removed lines) |
    new (added lines) |
    commit (commit headers) |
    whitespace (highlighting whitespace errors)

git-log --decorate output

git-log --decorate color customize <slot>

color.decorate.<slot> where <slot> is
    branch (local branches) | 
    remoteBranch (remote-tracking branches) | 
    tag (tags) | 
    stash (stash) | 
    HEAD (HEAD) |

git-grep

git-grep enable/disable color

color.grep:  always | false (or never) | auto (or true)

If true or auto, only use color when output is to the terminal. Defaults to false.

git-grep color customize <slot>

color.grep.<slot> where <slot> is
    context (non-matching text in context lines (when using -A, -B, or -C)) |
    filename (filename prefix (when not using -h)) |
    function (function name lines (when using -p)) |
    linenumber (line number prefix (when using -n)) |
    match (matching text (same as setting matchContext and matchSelected)) |
    matchContext (matching text in context lines) |
    matchSelected (matching text in selected lines) |
    selected (non-matching text in selected lines)
    separator (separators between fields on a line (:, -, and =) and between hunks (--))

git-add, git-clean --interactive

git-add, git-clean --interactive enable/disable for interactive prompts and displays

color.interactive:   always | false (or never) | auto (or true)

When set to true or auto, use colors only when the output is to the terminal. Defaults to false.

git.interactive color customize <slot>

color.interactive.<slot> where <slot> is
    prompt |
    header | 
    help | 
    error

git-status

git-status enable/disable color

color.status:  always | false (or never) | auto (or true)

If set to auto (or true) colors are used only when the output is to a terminal. Defaults to false.

git-status color customize <slot>

color.status.<slot> where <slot> is
    header (the header text of the status message) |
    added or updated (files which are added but not committed) |
    changed (files which are changed but not added in the index) |
    untracked (files which are not tracked by Git) |
    branch (the current branch) |
    nobranch (the color the no branch warning is shown in, defaulting to red)

git-show-branch

git-show-branch enable/disable color

color.showbranch: always | false (or never) | auto (or true)

If set to auto (or true) colors are used only when the output is to a terminal. Defaults to false.
